01 — Fundamentals

Catalog Structure

🛒

Catalog

Container that holds catalog items. Organizations create separate catalogs for IT, HR, Facilities, etc.

containerdepartment
sc_catalog
📂

Category

Groups items logically within a catalog — Hardware, Software, Access, Network — for easy navigation.

groupingnavigation
sc_category
📦

Catalog Item

A requestable service — "Laptop Request", "VPN Access", "Software Install". Contains variables for user input.

requestablevariables
sc_cat_item
📝

Variables

Input fields on catalog forms — text, dropdowns, checkboxes, date pickers. Capture what the user needs.

textselectreference
item_option_new
📋

Record Producer

Creates records directly on target tables (like Incident) instead of going through REQ/RITM flow.

direct createincident
sc_cat_item_producer
02 — Analogy

Amazon for Enterprise

🛒 Think of Service Catalog Like Amazon Shopping
Browse → Select → Order → Delivered — same flow, enterprise services
🏪
Catalog
Amazon Store
The overall store — IT Catalog, HR Catalog — like Amazon categories.
📦
Catalog Item
Product Listing
"Laptop Request" = a product you can order. Has options (color, size = variables).
🧾
REQ
Order Confirmation
Your overall order — may contain multiple items. One REQ per checkout.
📬
RITM
Individual Item
Each product in your order gets tracked separately. One RITM per item.
🚚
SCTASK
Delivery Task
The fulfillment work — shipping, setup, configuration. Assigned to teams.
User browses Catalog → Selects Item → Submits (REQ) → Each item tracked (RITM) → Teams fulfill (SCTASK) → Done ✅

05 — APIs

Catalog Scripting

query-ritm.js
// Query all open requested items
var gr = new GlideRecord('sc_req_item');
gr.addQuery('state', '!=', 3);
gr.query();
while (gr.next()) {
  gs.info(gr.number + ' - '
    + gr.cat_item.getDisplayValue());
}
access-variables.js
// Access catalog variables on RITM
var ritm = new GlideRecord('sc_req_item');
ritm.get('sys_id_here');

// Get variable value
var laptopType = ritm.variables.laptop_type;
var os = ritm.variables.operating_system;

gs.info('Laptop: ' + laptopType);
gs.info('OS: ' + os);
06 — Pro Tips

Catalog Tips

01

Separate Catalogs by Department

Create IT Catalog, HR Catalog, Facilities Catalog. Keeps items organized and access controlled.

02

Use Variable Sets

Reuse common variables across items. Don't duplicate "Delivery Location" on every single item.

03

Catalog UI Policies First

Use catalog UI policies for simple show/hide logic. Only write catalog client scripts for complex scenarios.

04

Flow Designer Over Workflow

Use Flow Designer for all new catalog automation. Legacy Workflow Editor is being deprecated.

05

Limit Variables

Too many variables confuse users. Keep forms simple — only ask what you truly need for fulfillment.

06

Meaningful Variable Names

Use vpn_access_required not var_001. Future developers will thank you.

07

Test in Service Portal

Catalog behavior differs between platform UI and Service Portal. Always test both views.

08

Validate Inputs

Add mandatory flags and catalog client scripts to prevent incomplete or invalid submissions.

09

Debug via RITM Table

Most debugging starts at sc_req_item. Check variables, state, and related tasks from there.

10

Monitor Request Performance

Use reports on sc_req_item to track fulfillment times, bottlenecks, and approval delays.

08 — Quiz

Test Yourself

0/5
Score
Q01 Easy
Which table stores Requested Items (RITM)?
A incident
B sc_request
C sc_req_item
D sc_task
✅ C — sc_req_item stores individual requested items (RITM). sc_request is the parent REQ. sc_task stores fulfillment tasks.
Q02 Easy
REQ represents:
A Overall Request (order)
B Individual Item
C Fulfillment Task
D Approval record
✅ A — REQ (sc_request) is the overall request/order. It contains one or more RITMs (individual items).
Q03 Medium
True or False: SCTASK is used for fulfillment tasks.
A True
B False
✅ A (True) — SCTASKs (sc_task) are fulfillment tasks assigned to teams — like "Ship laptop" or "Configure VPN".
Q04 Medium
Which component is the container for catalog items?
A Task
B Variable
C Workflow
D Catalog
✅ D — A Catalog (sc_catalog) is the top-level container. It holds categories, which hold catalog items.
Q05 Hard
Which tool automates catalog request fulfillment in modern ServiceNow?
A Flow Designer
B Dictionary
C UI Policy
D Script Include
✅ A — Flow Designer is the modern tool for automating catalog workflows — approvals, task creation, notifications, and integrations.
